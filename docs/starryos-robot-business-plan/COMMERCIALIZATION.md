## ⚠️ 风险与对策

### 主要风险识别

| 风险类型 | 风险描述 | 可能性 | 影响 | 对策 |
|---------|---------|--------|------|------|
| **技术风险** | AI模型效果不达预期 | 中 | 高 | 迭代优化、云端增强 |
| **市场风险** | 用户接受度不及预期 | 中 | 高 | 小批量验证、快速调整 |
| **竞争风险** | 大厂入局竞争 | 高 | 中 | 差异化定位、生态壁垒 |
| **供应链风险** | 芯片短缺或涨价 | 中 | 中 | 多供应商、库存储备 |
| **资金风险** | 融资不及时 | 低 | 高 | 控制成本、尽早营收 |
| **人才风险** | 核心成员流失 | 中 | 高 | 股权激励、文化建设 |

### 应急预案

```
场景 A: 产品延期
├─ 触发条件: 延期超过3个月
├─ 应对措施:
│   ├─ 砍掉非核心功能，保证最小可行版本
│   ├─ 与预购用户透明沟通
│   └─ 提供补偿方案
└─ 责任人: CTO

场景 B: 销量不及预期
├─ 触发条件: 连续3个月销量低于目标50%
├─ 应对措施:
│   ├─ 分析原因 (产品/价格/渠道)
│   ├─ 调整营销策略
│   ├─ 考虑降价促销
│   └─ 加强用户访谈，快速迭代
└─ 责任人: 销售VP

场景 C: 现金流紧张
├─ 触发条件: 现金储备低于3个月运营成本
├─ 应对措施:
│   ├─ 启动紧急融资
│   ├─ 削减非必要开支
│   ├─ 加速应收账款回收
│   └─ 寻求过桥贷款
└─ 责任人: CEO
```
